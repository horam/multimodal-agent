    CLI Usage (agent)

    The agent CLI is the main interface for interacting with the Multimodal-Agent system. It supports:

    • Text queries
    • Image + text queries
    • Interactive chat
    • RAG memory browsing
    • Project learning
    • Flutter code generation (v0.8.0+)
    • Config management
    • Running a FastAPI server

    Basic Commands

    Show version

    agent --version

    Ask a text question

    agent ask "What is Flutter?"

    JSON mode

    agent ask "example" --json

    Disable RAG for a single query

    agent ask "hello" --no-rag

    Image + Text Queries

    agent image cat.jpg "what is in this image?"

    Supports --json and --no-rag just like ask.

    Interactive Chat Mode

    agent chat

    With persistent session ID:

    agent chat --session my-dev-session

    Disable RAG during chat:

    agent chat --no-rag

    Flutter Code Generation (v0.8.0+)

    The CLI can generate Flutter boilerplate files: • Widgets → lib/widgets/ • Screens → lib/screens/ • Models → lib/models/

    Run inside a Flutter project (must contain pubspec.yaml).

    ⸻

    Generate a stateless widget

    agent gen widget MyWidget

    Generate a stateful widget

    agent gen widget Counter --stateful

    Generate a screen

    agent gen screen HomeScreen

    Generate a model

    agent gen model UserProfile

    Overwrite existing file

    agent gen widget MyWidget --override

    All names are auto-normalized:

    • Class: PascalCase
    • File: snake_case.dart

    Example:

    agent gen widget UserCard

    Creates:

    lib/widgets/user_card.dart

    History & Memory (RAG)

    Show recent memory

    agent history show

    Limit number of rows:

    agent history show --limit 20

    Hide noise

    agent history show --clean

    Delete all memory

    agent history clear

    Summarize memory via LLM

    agent history summary

    Delete specific memory item

    agent history delete 42

    Project Learning (RAG Profiles)

    Teach the agent about a project:

    agent learn-project /path/to/project

    Store with specific ID:

    agent learn-project /app --project-id myapp

    List learned projects:

    agent list-projects

    Show project profile:

    agent show-project myapp

    Inspect without saving:

    agent inspect-project /app

    Server Mode (FastAPI)

    Run a local API server:

    agent server --port 8000

    Config Management

    Set API key

    agent config set-key YOUR_KEY

    Set chat model

    agent config set-model gemini-2.5-flash

    Set image model

    agent config set-image-model gemini-2.5-flash

    Set embedding model

    agent config set-embed-model text-embedding-004

    Show full config

    agent config show

    Offline Fake Mode

    If GOOGLE_API_KEY is missing, the agent runs in fake offline mode.

    Example:

    agent ask "hello"

    Output:

    FAKE_RESPONSE: hello

    Useful for:

    • CI environments
    • Tests
    • No network access

    Quota & Rate Limits (Free Tier)

    When using the Gemini API free tier, you may encounter:

    • 429 RESOURCE_EXHAUSTED
    • Quota exceeded
    • Temporary request failures

    This is expected behavior and not a CLI bug.

    Notes: - The CLI talks directly to Gemini and may continue working even when the server or VS Code extension fails. - Requests may succeed intermittently until quota fully resets.

    What to do: - Wait for quota reset (usually within 24 hours) - Reduce request frequency - Switch to a lighter model - Upgrade your Gemini API plan

    Model Availability

    If you configure an unsupported or deprecated model, the CLI may fail with:

    • 404 NOT_FOUND (model)

    Fix: - Update your model via: 

    agent config set-model gemini-2.5-flash

    • Verify supported models at: https://ai.google.dev/gemini-api/docs/models

    Debug Logging

    Enable verbose logs:

    agent --debug ask "hi"
